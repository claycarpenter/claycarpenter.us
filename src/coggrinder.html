App Features
  • Fully local cache of tasks allows for offline editing and revert functionality.
  • Local changes can either be saved and pushed to the Google servers, or reverted and wiped out.
  • Refresh allows for fresh task data to be pulled from the Google servers.
  • Provides features not found in the Google Task web interfaces:
    • Manipulation (complete/uncomplete, hierarchy reorganization) of multiple tasks simultaneously.
    • Ability to view and edit multiple task lists concurrently.
    • All hierarchies are collapsible so that users can focus on only the task information they need to see at the moment.
  • Provides authentication through Google's OAuth service.
Development Highlights
  • Extensive unit test coverage.
  • Throughly documented, with nearly a line of documentation for each line of code written.
  • Implemented with an MVC architecture that combines controllers written in Python, GTK widgets serving as views, and model data provided by the Google Task services.
